    Operaciones numéricas vectoriales y estadísticas con NumPy

    1. Teoría

    En NumPy, casi todo gira alrededor del ndarray, un contenedor multidimensional de datos homogéneos (todos los elementos del mismo tipo) sobre el que se aplican operaciones vectorizadas muy rápidas. Normalmente lo creamos con np.array(...), que convierte listas u otras estructuras en un array NumPy.

    En esta píldora veremos un “mini-kit” de funciones que aparecen juntas en análisis numérico, series temporales y finanzas:

    • np.array: crea el array de partida a partir de listas u otras estructuras.
    • np.argsort: devuelve los índices que ordenarían un array, es decir, no ordena los datos directamente, sino que indica cómo reordenarlos.
    • np.corrcoef: calcula la matriz de correlaciones de Pearson entre una o varias series numéricas; devuelve valores entre -1 y 1 que indican la relación lineal entre variables.
    • np.cumsum / np.cumprod: cumsum devuelve la suma acumulada de los elementos de un array, mientras que cumprod devuelve el producto acumulado a lo largo de un eje. Son muy usados para series acumuladas (ventas, accidentes, patrimonio, etc.).
    • np.diff: calcula las diferencias discretas entre elementos consecutivos: por defecto out[i] = a[i+1] - a[i]. Es típico en series temporales para obtener cambios diarios, retornos, variaciones, etc.
    • np.histogram: calcula el histograma numérico de un conjunto de datos, segmentando los valores en “bins” y contando cuántas observaciones caen en cada intervalo.
    • np.log: aplica el logaritmo natural elemento a elemento; es la inversa de la exponencial, de forma que log(exp(x)) = x.
    • np.power: eleva elemento a elemento los valores de un array base a las potencias de otro array (o a un escalar), es decir, calcula x1**x2 con broadcasting.

    Estas funciones suelen aparecer en workflows del tipo: “creo un array de datos → lo transformo (log, diff, power, cum) → analizo su distribución (histogram) → calculo relaciones entre series (corrcoef)”.

    2. Ejemplo sintético con NumPy

    Imaginemos que tenemos precios diarios de dos activos financieros y queremos:

    • Calcular retornos simples y logarítmicos.
    • Obtener retornos acumulados.
    • Ver la distribución de retornos.
    • Medir la correlación entre los dos activos.
    • Localizar los peores días (según argsort).
    import numpy as np

# 1. Crear arrays de precios (datos sintéticos)
precios_a = np.array([100, 101, 103, 102, 105, 107, 106])
precios_b = np.array([50,  49,  51,  52,  50,  53,  54])

# 2. Retornos simples (%) usando diff
retornos_a = np.diff(precios_a) / precios_a[:-1]
retornos_b = np.diff(precios_b) / precios_b[:-1]

# 3. Retornos logarítmicos
log_retornos_a = np.log(precios_a[1:] / precios_a[:-1])
log_retornos_b = np.log(precios_b[1:] / precios_b[:-1])

# 4. Retorno acumulado (producto acumulado de (1+r) - 1)
ret_acum_a = np.cumprod(1 + retornos_a) - 1
ret_acum_b = np.cumprod(1 + retornos_b) - 1

# 5. Histograma de retornos (frecuencias por rango)
freqs_a, bins_a = np.histogram(retornos_a, bins=5)

# 6. Correlación entre las dos series de retornos
corr_mat = np.corrcoef(retornos_a, retornos_b)

# 7. Localizar peores días de retornos (menores retornos primero)
idx_orden_a = np.argsort(retornos_a)
peores_dias_a = idx_orden_a[:2]   # índices (0-based) de los 2 peores días

print("Retornos A:", retornos_a)
print("Retornos acumulados A:", ret_acum_a)
print("Histograma A (frecuencias):", freqs_a)
print("Matriz de correlaciones:")
print(corr_mat)
print("Peores días A (índices):", peores_dias_a)

    Si quisiéramos aplicar una transformación de potencia (por ejemplo, elevar los retornos al cuadrado para aproximar volatilidad), podemos usar np.power:

    # Volatilidad aproximada (retornos al cuadrado)
vol_approx_a = np.power(retornos_a, 2)

    Y un pequeño histograma con matplotlib:

    import matplotlib.pyplot as plt

plt.hist(retornos_a, bins=5)
plt.title("Histograma de retornos del activo A")
plt.xlabel("Retorno")
plt.ylabel("Frecuencia")
plt.tight_layout()
plt.show()

    3. Diagrama de flujo (Mermaid)

    flowchart TD A[Datos originales ej. precios diarios] --> B[np.array -> ndarray] B --> C[np.diff calcular variaciones] C --> D[np.log / np.power transformaciones matemáticas] D --> E[np.cumsum / np.cumprod acumulados] E --> F[np.histogram analizar distribución] E --> G[np.corrcoef correlación entre series] E --> H[np.argsort localizar máximos/mínimos]

    4. Aplicación al dataset de accidentes de NYC

    Ahora usamos el archivo NYC.csv (accidentes de tráfico) para ilustrar estos conceptos. En vez de precios, trabajaremos con número de accidentes por día:

    1. Leemos el CSV.
    2. Agrupamos por fecha para obtener un conteo diario.
    3. Convertimos la serie resultante en un np.array.
    4. Aplicamos diff, cumsum, histogram, log, etc.
    import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# 1. Cargar el dataset
df = pd.read_csv("data/NYC.csv")

# 2. Asegurarnos de que la fecha es datetime
df["FECHA_ACCIDENTE"] = pd.to_datetime(df["FECHA_ACCIDENTE"], errors="coerce")

# 3. Contar accidentes por día
acc_por_dia = (
    df
    .groupby("FECHA_ACCIDENTE")
    .size()
    .sort_index()          # nos asegura orden temporal
)

# 4. Pasar a array NumPy
acc_array = acc_por_dia.to_numpy()

# 5. Variación diaria de accidentes (diff)
variacion_diaria = np.diff(acc_array)

# 6. Accidentes acumulados a lo largo del tiempo (cumsum)
acc_acumulados = np.cumsum(acc_array)

# 7. Transformación logarítmica para suavizar grandes diferencias
#    (sumamos 1 para evitar log(0))
log_acc = np.log(acc_array + 1)

# 8. Histograma de accidentes diarios
freqs, bins = np.histogram(acc_array, bins=10)

print("Primeros días y accidentes:
", acc_por_dia.head())
print("Variación diaria:", variacion_diaria[:5])
print("Total acumulado último día:", acc_acumulados[-1])
print("Frecuencias del histograma:", freqs)

# 9. Histograma gráfico
plt.hist(acc_array, bins=10)
plt.title("Distribución de accidentes diarios en NYC")
plt.xlabel("Número de accidentes al día")
plt.ylabel("Frecuencia")
plt.tight_layout()
plt.show()

    Si queremos ver qué días son especialmente altos o bajos en número de accidentes, podemos usar argsort sobre el array:

    # Índices que ordenarían los días por número de accidentes
idx_orden = np.argsort(acc_array)

# Días con menos accidentes
dias_minimos = acc_por_dia.index[idx_orden[:5]]
valores_minimos = acc_array[idx_orden[:5]]

# Días con más accidentes
dias_maximos = acc_por_dia.index[idx_orden[-5:]]
valores_maximos = acc_array[idx_orden[-5:]]

print("Días con menos accidentes:")
for d, v in zip(dias_minimos, valores_minimos):
    print(d.date(), "->", v)

print("
Días con más accidentes:")
for d, v in zip(dias_maximos, valores_maximos):
    print(d.date(), "->", v)

    Y si queremos correlación entre, por ejemplo, accidentes en dos distritos distintos, podemos combinar pandas y NumPy:

    # Tabla con accidentes por día y distrito
tabla = (
    df
    .groupby(["FECHA_ACCIDENTE", "DISTRITO"])
    .size()
    .unstack("DISTRITO")
    .fillna(0)
)

# Convertir dos columnas (distritos) a arrays
bronx = tabla["BRONX"].to_numpy()
brooklyn = tabla["BROOKLYN"].to_numpy()

# Correlación entre ambas series temporales
corr_bx_bk = np.corrcoef(bronx, brooklyn)

print("Matriz de correlación BRONX vs BROOKLYN:")
print(corr_bx_bk)

    6. Conclusión

    Las operaciones numéricas vectoriales de NumPy permiten construir rápidamente pipelines del tipo:

    array crudo → transformaciones (diff, log, power) → acumulados (cumsum, cumprod) → distribución (histogram) → relaciones entre series (corrcoef) → ranking (argsort)

    Este flujo es fundamental para:

    • Analizar series temporales (financieras, accidentes, sensores…).
    • Estudiar distribuciones y detectar días “extremos”.
    • Medir relaciones entre diferentes variables o segmentos (por ejemplo, distritos).

    Cuando combinamos estas funciones con un DataFrame bien tipado y limpio en pandas (gracias a astype, categorías, fechas, etc.), podemos pasar de datos en bruto a indicadores e insights de forma compacta y eficiente, reutilizando siempre el mismo lenguaje vectorizado que caracteriza a NumPy y pandas.
