    Conversión de tipos con astype en pandas

    Teoría

    El análisis de datos no sólo consiste en manipular tablas, sino en asegurarse de que cada columna tiene un tipo adecuado. Las bases de datos y los ficheros CSV frecuentemente almacenan valores numéricos como cadenas de texto, fechas como cadenas de caracteres o códigos postales como números con decimales porque hay valores nulos. Trabajar con tipos incorrectos puede provocar errores en cálculos, malgastar memoria y complicar el trabajo con modelos de aprendizaje automático. La función astype de pandas permite forzar el tipo de una serie o un DataFrame para convertirlo a otro tipo. Según la documentación oficial, astype “convierte un objeto de pandas a un tipo de datos especificado” y admite pasar un tipo concreto o un diccionario para convertir varias columnas a tipos distintos. La función tiene tres argumentos principales:

    • dtype – tipo al que se quiere convertir (por ejemplo 'int', 'float', 'str', 'category' o tipos de numpy).
    • copy – si se establece en True devuelve una copia con el nuevo tipo; con False modifica el objeto original.
    • errors – controla si se muestran errores ('raise') o se ignoran ('ignore').

    El blog “Understanding pandas astype()” explica de forma más divulgativa que astype() es como “cambiar el traje de tus datos”: la información es la misma, pero adaptada a lo que necesitas. Esta conversión es útil para optimizar memoria (por ejemplo, transformando textos repetidos en tipos category) o para preparar datos para modelos que exigen tipos numéricos. Un aspecto importante es que, al convertir cadenas a enteros, los decimales se truncan (no se redondean).

    Ejemplo sintético

    El siguiente ejemplo crea un DataFrame con dos columnas: una categoría y un precio almacenado como cadena. Usamos astype para convertir store_location a tipo category y price a entero.

    import pandas as pd

Df = pd.DataFrame({
    'store_location': ['A', 'B', 'A'],
    'price': ['10', '20', '30']
})

# Conversión de tipo
Df['store_location'] = Df['store_location'].astype('category')
Df['price'] = Df['price'].astype(int)

print(Df.dtypes)
print(Df)

    La salida muestra que store_location pasa a ser una categoría y price un entero, y que los valores se han convertido correctamente:

    store_location    category
price                int64
dtype: object
  store_location  price
0              A     10
1              B     20
2              A     30

    Aplicación al dataset de accidentes de NYC

    El archivo NYC.csv contiene 30 columnas con información sobre accidentes de tráfico en la ciudad de Nueva York (fecha, hora, distrito, localización, número de heridos/fallecidos, tipo de vehículo, etc.). Al cargar el archivo con pandas, muchas columnas aparecen como object (cadenas) aunque en realidad son fechas, códigos postales o categorías. A continuación se muestra cómo leer el dataset y convertir algunas columnas clave:

    import pandas as pd

# Leer el CSV
df = pd.read_csv('/home/oai/share/NYC.csv')

# Conversión de tipos
df['FECHA_ACCIDENTE'] = pd.to_datetime(df['FECHA_ACCIDENTE'], format='%Y-%m-%d', errors='coerce')
df['DISTRITO'] = df['DISTRITO'].astype('category')
df['CODIGO_POSTAL'] = df['CODIGO_POSTAL'].astype('Int64')  # permite valores nulos
df['TIPO_VEHICULO_1'] = df['TIPO_VEHICULO_1'].astype('category')

# Mostrar los nuevos tipos
print(df[['FECHA_ACCIDENTE', 'DISTRITO', 'CODIGO_POSTAL', 'TIPO_VEHICULO_1']].dtypes)

# Calcular memoria antes y después de la conversión (MB)
mem_before = pd.read_csv('/home/oai/share/NYC.csv').memory_usage(deep=True).sum() / 1024**2
mem_after = df.memory_usage(deep=True).sum() / 1024**2
print(f"Memoria antes de convertir: {mem_before:.2f} MB")
print(f"Memoria después de convertir: {mem_after:.2f} MB")

# Contar accidentes por distrito
acc_by_district = df.groupby('DISTRITO').size().sort_values(ascending=False)
print(acc_by_district.head())

    En este caso:

    • La columna FECHA_ACCIDENTE se transforma en datetime64[ns] para poder extraer año, mes o día y realizar análisis temporal.
    • DISTRITO y TIPO_VEHICULO_1 se convierten en categorías, lo que reduce el consumo de memoria y facilita agrupaciones.
    • CODIGO_POSTAL se pasa a Int64, que admite valores nulos (<NA>). Convertirlo directamente a int habría generado errores en filas con código postal ausente.

    Al medir la memoria del DataFrame completo, se observa que pasa de aproximadamente 71,39 MB a 58,47 MB tras la conversión, un ahorro notable cuando se trabaja con grandes volúmenes de datos.

    Visualización: accidentes por distrito

    La conversión de DISTRITO a categoría nos permite agrupar rápidamente y visualizar los accidentes por distrito. El siguiente gráfico muestra los cinco distritos con mayor número de accidentes registrados en el archivo:

    import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# Cargar el conjunto de datos
df = pd.read_csv('data/NYC.csv')

# Contar accidentes por distrito (top‑10)
counts = df['DISTRITO'].value_counts().head(10)

# Crear el gráfico de barras
plt.figure(figsize=(8, 4))
sns.barplot(x=counts.index, y=counts.values, palette='viridis')
plt.xticks(rotation=45)
plt.title('Top distritos con más accidentes en NYC')
plt.xlabel('Distrito')
plt.ylabel('Número de accidentes')
plt.tight_layout()


# Mostrar la figura en pantalla (opcional)
plt.show()

    Actividades y preguntas

    1. Profundizar en la conversión de fechas y horas. Convierte la columna HORA_ACCIDENTE a un objeto de tipo hora (datetime.time) e investiga en qué franjas horarias se concentran más accidentes.
    2. Crear nuevas columnas. A partir de FECHA_ACCIDENTE, genera variables como el año, mes o día de la semana y analiza tendencias estacionales.
    3. Optimización de memoria. Calcula el consumo de memoria del DataFrame antes y después de convertir todas las columnas de tipo object que tengan pocos valores distintos a category. ¿Cuánto se ahorra?
    4. Conversión segura de variables numéricas. Algunas columnas de número de heridos/fallecidos ya son enteros (int64). Introduce a propósito un valor no numérico (p.ej. 'dos') en una copia del DataFrame y comprueba la diferencia entre usar astype(int) (genera error) y pd.to_numeric(..., errors='coerce') (convierte los errores en NaN).
    5. Análisis por tipo de vehículo. Tras convertir TIPO_VEHICULO_1 a categoría, agrupa los accidentes por tipo de vehículo y estudia qué vehículos están implicados en más accidentes con heridos.

    Conclusión y aplicaciones generales

    La función astype de pandas es una herramienta esencial para asegurar la coherencia de tipos en un conjunto de datos. Permite convertir columnas individuales o todo un DataFrame a tipos concretos, ya sea para optimizar la memoria, realizar cálculos correctamente o preparar variables para algoritmos de machine learning. La documentación oficial indica que se puede pasar un único tipo o un mapeo de columnas a tipos, y la conversión puede hacerse sin copiar el objeto original si se desea. Artículos divulgativos destacan que esta función es especialmente útil al trabajar con datos reales, donde los números suelen venir como cadenas o conviene pasar textos repetidos a categorías para ahorrar recursos.

    En el dataset de accidentes de NYC se observa cómo convertir columnas clave (fecha, distrito, código postal y tipo de vehículo) facilita análisis temporales, agrupaciones y visualizaciones. Además, la conversión de columnas con muchas repeticiones a category reduce significativamente el uso de memoria, lo que permite trabajar con conjuntos de datos grandes en equipos con recursos limitados. Estas mismas técnicas se aplican a cualquier proyecto de análisis de datos: desde preparar un dataset para un modelo predictivo hasta limpiar y comprimir datos antes de almacenarlos.
